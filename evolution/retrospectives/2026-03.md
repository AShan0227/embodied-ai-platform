## 5. SOUL.md 调整建议（需 Board 批准）

### 建议 1：为 codex 添加"外部依赖超时"行为

**问题：** PET-8 共 14 轮续接，其中大部分是在无外部 tracker 工具可用的情况下反复审计交接文档是否过期。

**建议调整（SOUL.md 新增段落）：**

```
## 外部依赖超时规则

当任务在 repo 层面的验收标准已全部满足，但外部工具（如 Linear、Slack）不可用时：
- 最多再追加 1 轮交接文档更新
- 之后强制停止，并在交付物中注明："外部闭环需人工操作：[具体步骤]"
- 不再进行"检查是否有新遗漏"的循环审计
```

**预期效果：** 将 tracker 阻塞任务总轮次从 14+ 压缩到 ≤ 4 轮。

**Board 批准需求：** ☐ approved / ☐ rejected

### 建议 2：为 codex 添加"CI 根因优先"原则

**问题：** PET-13 CI 调试花了 4 轮才定位到 `python -m pytest` vs `pytest` 入口差异。

**建议调整（SOUL.md 新增段落）：**

```
## CI 调试优先顺序

遇到 CI 失败时，按以下顺序排查：
1. 立即抓取原始日志（`gh run view --log-failed` 或 `gh api`），不依赖 step outcome 元数据
2. 检查 Python 版本与 pytest 入口方式的一致性
3. 跳过工作流逻辑推理，直接看实际报错
```

**Board 批准需求：** ☐ approved / ☐ rejected
